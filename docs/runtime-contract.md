# Runtime Contract (Markdown → DOM → Client JS)

The contract between the site skeleton (Astro / remark / Shiki) and the client
runtime (`src/client/`).

## Canonical Markdown notation → build output

| Notation | Output DOM |
|---|---|
| ```` ```python ```` | `pre.astro-code` (static Shiki highlighting only) |
| ```` ```python file=X.py ```` | `pre[data-file="X.py"]` (file tab rendered via CSS `::before`) |
| ```` ```python exec file=X.py [mode=append] ```` | `pre[data-exec][data-mode][data-file]` |
| ```` ```python noexec ```` | explicit opt-out from exec detection (resolved at import time) |
| ```` ```text output ```` | `pre[data-sample-output]` (expected output) |
| `::widget{name="NAME"}` | `div.widget-mount[data-widget="NAME"]` (children = non-JS fallback) |

The transforms live in `src/plugins/shiki-exec-transformer.mjs` and
`src/plugins/remark-widget-directive.mjs`.
Classes are avoided because they collide with Astro's internal `astro-code`
class handling — **data attributes are the only contract**.

## Client side (src/client/entry.ts)

- The chapter slug comes from the `main[data-chapter]` attribute
- `pre[data-sample-output]` → wrapped into `<details class="sample-output-details">`
- `pre[data-exec]` → CodeMirror 6 editor + Run/Reset + output panel (`exec-block/`)
  - The original code is taken from `pre.textContent`
  - Run = exec in a shared Python namespace on the Worker (**no FS writes**;
    the chapter snapshot is mounted in its complete state at boot, so imports
    always succeed)
- `div.widget-mount[data-widget]` → dynamically imported from
  `widgets/registry.ts` and mounted (lazy-loaded via IntersectionObserver just
  before the widget scrolls into view)

## Pyodide Worker (runtime/)

- Pyodide itself is served from the jsDelivr CDN; the version is kept in sync
  with the npm `pyodide` package
- Boot: load numpy → mount the chapter's .py files from
  `public/code/manifest.json` into the FS → run bootstrap.py
- Packages are lazy-loaded by scanning import statements
  (numpy / matplotlib / pillow). When matplotlib loads for the first time,
  `public/fonts/NotoSansJP-Regular.ttf` is registered so Japanese labels render
- Training runs through the `train-start` RPC as chunked execution (report
  progress and check the cancel flag every n steps). GitHub Pages cannot set
  COOP/COEP headers, so SharedArrayBuffer is unavailable — chunking is the only
  soft-cancel mechanism
- Recovery from runaway code is `worker.terminate()` + reboot (the "reset
  environment" button on the status pill)

## Data assets

- `public/code/chapter-NN/*.py` + `manifest.json` — generated by `pnpm import:content`
- `public/data/ch5/` — target.png + precomputed.json (`scripts/gen-ch5-precomputed.py`)
- `public/data/ch9/` — cameras.json + test-views/ + trained-params.json (`scripts/train-ch9-export.py`)
- All client fetches are rooted at `import.meta.env.BASE_URL` (base-path safe)

## Publishing a new chapter

1. If needed, add `::widget{name}` lines or fence meta (`noexec`, etc.) to the
   source markdown (the local content workspace)
2. Implement widgets in `src/client/widgets/chN-*.ts` and register them in
   `registry.ts`
3. `pnpm import:content -- --chapter NN`
4. Flip the chapter's `status` to `'published'` in `src/data/chapters.ts`
5. Verify with `pnpm build && pnpm preview`, then commit
